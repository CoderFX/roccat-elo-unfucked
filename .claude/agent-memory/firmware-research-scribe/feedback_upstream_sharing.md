---
name: Feedback — Upstream Sharing Rules
description: What must never appear in public commits, upstream PRs, or issues for this project
type: feedback
---

NEVER include in upstream PRs, issues, or public commits:
- Claude/AI attribution or mentions
- ROM addresses, firmware disassembly, or function maps
- Firmware patch layers, frame padding, or attack tooling details
- Any RE methodology that could be viewed as circumventing security

**Why:** CLAUDE.md upstream contribution rules; legal and community norms in open-source RE projects.

**How to apply:** Before drafting anything for public sharing (HeadsetControl PR, eruption issue, etc.), strip all of the above. Draft for user review before posting.
